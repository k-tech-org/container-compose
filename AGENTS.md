# Repository Instructions

- Do not commit user-specific local paths, usernames, machine names, real customer
  project names, or values derived from them.
- Use generic fixture names such as `sample-app`, `example-service`, or
  `demo-project` in tests and documentation.
- When an absolute path is necessary in documentation, use a placeholder such as
  `/path/to/container-compose` or `$PWD`.
- Before release-oriented changes, search for local path fragments and
  project-specific names to ensure they are not present in tracked files.
